Wondering what it takes to become an amazing programmer?

Step one: breathe a sigh of relief, because you don't have to know everything!

To be a successful programmer, you don't have to be a walking database, or someone with a photographic memory, or Mark Zuckerberg. You just have to commit to a generous application of curiosity and persistence.

Not sure what code to use? Google it.

Not sure why your html file isn't displaying correctly in a browser? Start digging through your code for clues.

Don't know how to use a new tool that just became available? Ask your fellow coders in an online community.

The point is, a programming language is not so much a finite body of knowledge to be mastered as a set of guidlines to become familiar with while maintaining an attitude of "I can figure this out."
